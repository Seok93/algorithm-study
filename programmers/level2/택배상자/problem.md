### 🔍 문제 링크
[Level2 **택배 상자** 문제](https://school.programmers.co.kr/learn/courses/30/lessons/131704)

### 📘 문제 설명
영재는 택배상자를 트럭에 싣는 일을 합니다. 영재가 실어야 하는 택배상자는 크기가 모두 같으며 1번 상자부터 n번 상자까지 번호가 증가하는 순서대로 컨테이너 벨트에 일렬로 놓여 영재에게 전달됩니다. 컨테이너 벨트는 한 방향으로만 진행이 가능해서 벨트에 놓인 순서대로(1번 상자부터) 상자를 내릴 수 있습니다. 하지만 컨테이너 벨트에 놓인 순서대로 택배상자를 내려 바로 트럭에 싣게 되면 택배 기사님이 배달하는 순서와 택배상자가 실려 있는 순서가 맞지 않아 배달에 차질이 생깁니다. 따라서 택배 기사님이 미리 알려준 순서에 맞게 영재가 택배상자를 실어야 합니다.

만약 컨테이너 벨트의 맨 앞에 놓인 상자가 현재 트럭에 실어야 하는 순서가 아니라면 그 상자를 트럭에 실을 순서가 될 때까지 잠시 다른 곳에 보관해야 합니다. 하지만 고객의 물건을 함부로 땅에 둘 수 없어 보조 컨테이너 벨트를 추가로 설치하였습니다. 보조 컨테이너 벨트는 앞 뒤로 이동이 가능하지만 입구 외에 다른 면이 막혀 있어서 맨 앞의 상자만 뺄 수 있습니다(즉, 가장 마지막에 보조 컨테이너 벨트에 보관한 상자부터 꺼내게 됩니다). 보조 컨테이너 벨트를 이용해도 기사님이 원하는 순서대로 상자를 싣지 못 한다면, 더 이상 상자를 싣지 않습니다.

예를 들어, 영재가 5개의 상자를 실어야 하며, 택배 기사님이 알려준 순서가 기존의 컨테이너 벨트에 네 번째, 세 번째, 첫 번째, 두 번째, 다섯 번째 놓인 택배상자 순서인 경우, 영재는 우선 첫 번째, 두 번째, 세 번째 상자를 보조 컨테이너 벨트에 보관합니다. 그 후 네 번째 상자를 트럭에 싣고 보조 컨테이너 벨트에서 세 번째 상자 빼서 트럭에싣습니다. 다음으로 첫 번째 상자를 실어야 하지만 보조 컨테이너 벨트에서는 두 번째 상자를, 기존의 컨테이너 벨트에는 다섯 번째 상자를 꺼낼 수 있기 때문에 더이상의 상자는 실을 수 없습니다. 따라서 트럭에는 2개의 상자만 실리게 됩니다.

택배 기사님이 원하는 상자 순서를 나타내는 정수 배열 `order`가 주어졌을 때, 영재가 몇 개의 상자를 실을 수 있는지 return 하는 solution 함수를 완성하세요.

### 📕 제한사항
- 1 ≤ `order`의 길이 ≤ 1,000,000
- `order`는 1이상 `order`의 길이 이하의 모든 정수가 한번씩 등장합니다.
- `order[i]`는 기존의 컨테이너 벨트에 `order[i]`번째 상자를 i+1번째로 트럭에 실어야 함을 의미합니다.

### 📙 입출력 예
|order|result|
|:---|:---|
|[4, 3, 1, 2, 5]|2|
|[5, 4, 3, 2, 1]|5|

### 📒 입출력 예 설명
**입출력 예 #1**  
문제 예시와 같습니다.

**입출력 예 #2**  
모든 상자를 보조 컨테이너 벨트에 모두 넣고, 역순으로 하나씩 빼서 트럭에 싣습니다.

### 📔 나의 알고리즘 순서
1. 택배 기사가 요구하는 상자의 번호를 가져온다.
2. 메인 컨테이너 벨트와 보조 컨테이너 벨트 중 어디에 속하는지 확인한다.
3. 대상 컨테이너 벨트에서 꺼내올 수 있는지 확인한다.  
  3-1. 꺼낼 수 있다면, 넣어주고 1~3번 반복  
  3-2. 꺼낼 수 없다면, 택배 싣기 종료  

### ✅ 나의 해답코드
```javascript
function solution(order) {
  const main = Array.from({ length: order.length }, (_, i) => i + 1).sort((a, b) => b - a);
  const sub = [];

  let count = 0;
  let now = main[main.length - 1];
  for (const num of order) {
    if (num < now) {
      const prev = sub.pop();
      if (num !== prev) break;
      count++;
      continue;
    }

    while (main.length && main[main.length - 1] !== num) {
      sub.push(main.pop());
    }
    main.pop();
    now = main[main.length - 1];
    count++;
  }

  return count;
}
```

### ✨ 깔끔한 해답코드
```javascript
function solution(order) {
  let result = 0;
  const stack = [];

  for (let i = 1; i <= order.length; i++) {
    stack.push(i);

    while (stack.length !== 0 && stack.at(-1) === order[result]) {
      stack.pop();
      result++;
    }
  }

  return result;
}
```

### 🤔고민한점 & 💡배운점
1\) 🤔 메인 컨테이너 벨트는 큐처럼 동작하고, 보조 컨테이너 벨트는 스택처럼 동작함을 지문을 통해 알 수 있다. 택배 기사가 원하는 물품이 나올 때까지 메인 컨테이너 벨트를 순차적으로 진행시키고, 이때 나오는 물품은 보조 컨테이너 벨트에 옮겨두면 된다고 생각했다. 이후 택배 기사가 원하는 물품이 보조 컨테이너 벨트에 있다면, 제일 마지막에 넣은 택배를 꺼내와서 매칭해보는 방식으로 구현했다. 로직을 분리하고 구현하는 것 자체는 어렵지 않았다.

2\) 💡이후에 다른 사람들의 풀이를 보다가 깔끔한 해답 코드를 발견했다. 문제 내용상 보조 컨테이너 벨트에 옮기면 다시 메인 컨테이너 벨트로 옮길 필요가 없다. 메인 컨테이너의 택배 물품을 순차적으로 꺼내서 보조 컨테이너로 옮기고, 옮길 때마다 방금 넣은 물품이 택배기사가 원하는 물품인지 확인하면 됐다. 이때 원하는 물품이라면 보조 컨테이너에서 스택의 특성으로 이전에 넣은 물품을 바로 꺼내줄 수 있다. 

정리하자면, 굳이 메인 컨테이너 벨트, 보조 컨테이너 벨트를 나눠서 택배 기사가 원하는 물품을 찾을 필요가 없다. 어차피 원하는 물품이 아니라면 순차적으로 보조 컨테이너 벨트로 이동하기 때문에, 메인 컨테이너에서 물품을 하나 꺼내서 옮기는 작업은 확정적으로 진행하면서, 옮길 때마다 택배 기사가 해당 물품이 찾는 물품인지 확인만 하면 된다.
